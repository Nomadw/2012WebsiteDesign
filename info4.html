Security Issues and Measures

Amazon must hold customer's data under the Data Protection Act. This means that:
-All data must be processed fairly and lawfully.
-Data must only be obtained for specified lawful purposes, and not used for other reasons.
-Data kept must be adequate, relevant and not excessive.
-Data must be kept accurate and up-to-date.
-Data cannot be held for longer than necessary for the purpose.
-Data must be processed within the rights of the data subject under this act.
-The data held must be kept secure from other sources attempting to obtain the data.
-The data cannot be transferred to country or territory where adequate data protection laws
are not present.


Security measure Description
CAPTCHA Provides a visualisation of one or two words which you need to re-enter into a box. Prevents bots from registering and posting. Main security measure on anonymous posting boards such as 4chan.
SSL Encryption Encrypts information to send to a server. The server then requests they key for the encryption, which is sent in an encryption that the server holds the key for.
Password strength testing During registration a small program measures the strength of a password, which can help the client decide on how to build a stronger password to prevent hackers guessing the password.
IP blocking Blocks the IP of a user that may be trying to attack a server to prevent further attacks from them.
File prevention Only allows certain filetypes to be uploaded to a server, preventing users from uploading things such as scripts to access restricted parts of a server.
IP logging Keeps a log of which IP uploads and accesses content on the server, allowing attacking IPs to be found and blocked.
Forgotten password systems If a user forgets their password, this resets that users password and send the new password to their email account, to stop any user from being able to reset and log into anyone else's account.
